# AI-Ready Data 용어집

이 용어집은 프로젝트 안의 일관된 의미를 정의한다. 국제 표준의 공식 번역을 대체하지
않으며, 조직 용어는 [업무 용어사전 템플릿](../templates/glossary-template.md)으로
별도 관리한다.

| 용어 | 이 위키의 정의 |
| --- | --- |
| AI-Ready Data | 특정 AI 유즈케이스에 적합하며 발견·접근·해석·추적·보호·평가·갱신 가능한 데이터 |
| ACL | Access Control List. 사용자·그룹이 자원에 할 수 있는 행동을 나타내는 접근 목록 |
| ASR | Automatic Speech Recognition. 음성을 텍스트로 전사하는 기술 |
| Chunk / 청크 | 검색과 모델 컨텍스트에 사용할 자기완결적인 문서 조각 |
| Data lineage / 계보 | 원천이 어떤 변환·승인·버전을 거쳐 파생물과 답변이 됐는지의 연결 |
| Data product | 명확한 사용자·목적·소유자·품질·접근·생애주기를 가진 재사용 가능한 데이터 묶음 |
| Data steward | 데이터 목록·정의·품질·변경을 일상적으로 운영하는 역할 |
| DRM | Digital Rights Management. 문서 열람·복사·인쇄 등을 통제하는 기술과 정책 |
| EDM | Electronic Document Management. 전자문서 저장·버전·검색·워크플로를 지원하는 체계 |
| Embedding / 임베딩 | 텍스트·이미지 등의 의미적 특징을 수치 벡터로 표현한 것 |
| Golden set / 골든셋 | 질문, 정답/기대 동작, 허용 근거, 권한 조건을 사람이 승인한 평가 데이터 |
| Metadata / 메타데이터 | 제목, 소유자, 상태, 효력, 등급, ACL, 출처처럼 데이터의 의미와 관리를 설명하는 데이터 |
| OCR | Optical Character Recognition. 스캔·이미지의 글자를 텍스트로 인식하는 기술 |
| Provenance / 출처 이력 | 데이터가 어디서 누구에 의해 언제 생성·수집·변환됐는지의 증거 |
| RAG | Retrieval-Augmented Generation. 검색한 근거를 모델 컨텍스트에 제공해 답변하는 구조 |
| SLO | Service Level Objective. 품질·전파시간·가용성 등 합의된 서비스 목표 |
| SSoT | Single Source of Truth. 특정 질문·범위·시점에서 권위 있다고 승인된 원천과 판정 규칙 |
| Tombstone | 삭제 또는 비활성 상태를 다른 저장소에 전파하기 위한 표시·이벤트 |
| Vector database | 임베딩 벡터의 근접 검색을 지원하는 저장·검색 계층 |
| 데이터 거버넌스 | 데이터 결정을 누가 어떤 근거와 절차로 내리고 책임지는지 정한 운영체계 |
| 데이터 품질 | 특정 목적에서 데이터가 요구를 충족하는 정도. 정확성만을 뜻하지 않음 |
| 멱등성 | 같은 처리를 반복해도 결과 상태가 달라지지 않는 성질 |
| 원본 | 업무·법적 의미를 가진 최초 또는 공식 파일·레코드. 파생 텍스트와 구분함 |
| 파생물 | 원본에서 추출·정규화·청킹·요약·임베딩해 만든 재생성 가능한 표현 |

AI·ML 데이터 품질의 표준 용어와 생애주기 관점은
[ISO/IEC 5259-1:2024](https://www.iso.org/standard/81088.html)을 참고한다.
