# Office·PDF·Excel 정돈 플레이북

목표는 파일을 없애는 것이 아니라 **원본의 의미·구조·권한·버전을 잃지 않는 파생물**을
만드는 것이다. [Apache Tika](https://tika.apache.org/)처럼 다양한 파일 형식에서
텍스트와 메타데이터를 추출하는 도구를 사용할 수 있지만, 자동 추출 결과는 원본과
별개의 검증 대상이다.

::: info 이 페이지의 위치
이 플레이북은 **Data Ready**를 만든다. 텍스트·표·메타데이터를 정확히 추출한 뒤,
현재 효력과 업무 의미는 [Knowledge Ready](../knowledge-ready/)에서 별도로 승인한다.
:::

## 공통 원칙: 원본, 파생물, 서비스 표현을 나눈다

| 계층 | 예 | 변경 원칙 |
| --- | --- | --- |
| 원본 보존본 | `.docx`, `.pptx`, `.xlsx`, 서명 PDF | 해시와 ACL을 붙여 변경하지 않음 |
| 정규화 파생물 | Markdown/HTML, 표 CSV, 이미지, OCR JSON | 도구·버전·변환시각·원본 ID 기록 |
| AI 서비스 표현 | 청크, 요약, 키워드, 임베딩 | 언제든 재생성 가능, 원본 삭제를 상속 |

원본을 Markdown으로 덮어쓰지 않는다. 암호, DRM, 매크로, 임베디드 개체, 숨은 시트와
주석처럼 변환 도구가 놓칠 요소를 먼저 조사한다.

## 형식별 추출 계약

### Word와 한글 문서

- 제목 스타일, 문단, 목록, 표, 각주, 머리말·꼬리말을 분리한다.
- 변경 추적과 주석을 본문 확정 내용과 섞지 않는다.
- 이미지에는 페이지·문단 위치와 사람이 확인할 설명을 연결한다.
- 표가 페이지를 넘어가도 헤더와 병합 셀의 의미를 보존한다.

### PPT

- 슬라이드 제목, 본문, 도형 텍스트, 표, 발표자 노트를 구분한다.
- 화면 좌표만으로 읽는 순서를 단정하지 말고 슬라이드별 검수한다.
- 차트는 제목, 축, 단위, 범례, 핵심 수치와 결론을 구조화한다.
- 장식 이미지는 제외하고 업무 의미가 있는 도식은 설명 텍스트를 만든다.

### PDF와 스캔

- 텍스트 PDF인지 이미지 스캔인지 페이지별로 판별한다.
- OCR 언어, 해상도, 기울기 보정, 신뢰도와 페이지 좌표를 기록한다.
- 도장·서명·수기·표·다단 편집은 별도 검토 대상으로 표시한다.
- 전자서명과 원본 무결성 검증 결과는 텍스트 추출과 분리해 보존한다.

### Excel

Excel은 문서이면서 작은 애플리케이션이다. 보이는 셀 값만 CSV로 저장하면 수식,
단위, 숨은 시트, 병합 셀, 이름 정의, 오류 코드, 피벗, 매크로의 의미를 잃는다.

1. 통합문서 수준의 목적·소유자·기준일·버전을 기록한다.
2. 시트별 역할을 원천/계산/코드표/출력/임시로 분류한다.
3. 숨은 행·열·시트, 보호, 외부 링크, 매크로를 검사한다.
4. 헤더, 데이터 시작행, 기본키 후보, 단위, 결측 표현을 찾는다.
5. 수식과 계산 결과를 함께 보존하고 오류·순환참조를 보고한다.
6. 표마다 데이터 사전과 품질 규칙을 만들고 CSV/Parquet 등으로 파생한다.

[Apache Tika Office 메타데이터](https://tika.apache.org/3.2.3/api/org/apache/tika/metadata/Office.html)는
작성자·생성/저장일뿐 아니라 숨은 행·열·시트, 보호된 워크시트와 같은 속성도 다룬다.
도구가 지원하는 속성과 조직이 실제로 검사하는 속성을 혼동하지 않는다.

## 변환 후 검증

### 자동 검사

- 파일 수, 페이지/슬라이드/시트 수와 추출 성공 수
- 빈 본문, 비정상 문자, OCR 저신뢰 구간, 잘린 표
- 제목·목록 계층, 표 행·열 수, 이미지·첨부 누락
- 날짜·숫자·단위·코드 분포와 이상치
- 원본 ID·해시·ACL·변환 도구·버전 필드 존재

### 사람 표본 검사

파일 형식·부서·연도·보안등급별로 표본을 뽑고 반드시 어려운 사례를 포함한다.

- 표와 수식의 핵심 수치가 같은가?
- 슬라이드의 결론과 도식 관계가 살아 있는가?
- OCR이 사람 이름, 설비 ID, 단위와 소수점을 바꾸지 않았는가?
- 주석·초안·폐기 내용이 승인 본문으로 섞이지 않았는가?
- 원문으로 돌아갈 페이지/슬라이드/셀 위치가 있는가?

## 표준 메타데이터 예시

```yaml
source_id: edm:OPS-MNT-014:2.1
source_path: /approved/maintenance/OPS-MNT-014_v2.1.pdf
content_hash: sha256:<원본에서 계산한 값>
document_type: procedure
title: 설비 정기점검 절차
owner: 설비기술팀
status: approved
effective_from: 2026-07-01
classification: internal
acl_groups: [maintenance-engineers]
extraction:
  tool: approved-parser
  version: pinned-version
  extracted_at: 2026-07-22T10:00:00+09:00
  quality_flags: [table-review-required]
```

실제 해시는 자동으로 계산하며 예시 문자열을 복사하지 않는다.

## 하지 말아야 할 자동화

- 오래된 파일을 수정일만 보고 삭제하기
- AI 요약을 원본 또는 승인 문서로 승격하기
- 모든 Excel의 첫 행을 헤더로 가정하기
- DRM·암호 해제를 위해 승인되지 않은 우회 도구 사용하기
- 추출 실패 파일을 조용히 건너뛰기
- 문서 ACL 없이 중앙 벡터DB에 적재하기

변환 실패율을 품질 지표로 공개하고, 실패 원본은 격리 대기열에서 소유자에게 돌려준다.
다음은 [수집·정제·색인 파이프라인](../05-pipeline/ingestion.md)에서 이 계약을 반복
가능하게 만드는 단계다.
