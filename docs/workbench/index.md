# 읽는 대신, 첫 90일을 만드세요

가장 닮은 조직 상황을 고르고 10개 문항을 **실제 증거 기준**으로 평가한다. 결과는
준비 점수로 끝나지 않고 지금 승인할 세 과제, 책임자, 확인 지표, 30·60·90일 실행
순서로 이어진다.

::: tip 15분 워크숍으로 쓰는 법
현업 소유자, 데이터·IT, 보안 담당자가 같은 화면을 보되 각자 먼저 답한 뒤 차이가 큰
문항부터 토론한다. 평균 점수보다 “증거 있음”을 뒷받침할 문서·로그·승인 기록이
있는지가 중요하다.
:::

## 결과를 실행으로 바꾸는 규칙

- 상위 세 과제에는 역할명이 아닌 실제 책임자와 승인일을 적는다.
- Data Ready 또는 Knowledge Ready의 중단 조건이 남으면 AI 파일럿 범위를 넓히지 않는다.
- 30일마다 같은 문항을 다시 평가하고 점수보다 새로 확보한 증거를 비교한다.
- 인쇄 결과는 결정의 출발점이다. 실제 백로그는
  [90일 백로그·RACI](../templates/roadmap-raci.md)에 관리한다.
- 조직명, 시스템 주소, 문서 내용, 개인정보는 이 공개 페이지에 입력하지 않는다.

워크벤치의 게이트는 [NIST AI RMF Core](https://airc.nist.gov/airmf-resources/airmf/5-sec-core/)의
Govern·Map·Measure·Manage 활동을 작은 유즈케이스에 적용하도록 구성했다. 규제 준수나
보안 승인을 대신하지 않는다.
